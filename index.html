What is the difference between "null" and "undefined" in JavaScript?

Answer: "null" is an intentional absence of any value, while "undefined" means a variable has been declared but has not been assigned a value.

What's the difference between "let," "var," and "const" when declaring variables?

Answer: "let" and "const" are block-scoped, while "var" is function-scoped. "const" is used for constant values, while "let" and "var" can be reassigned.

Explain the concept of hoisting in JavaScript.

Answer: Hoisting is a JavaScript behavior where variable and function declarations are moved to the top of their containing scope during compilation, making them accessible before they're declared.

What is a closure in JavaScript?

Answer: A closure is a function that has access to its own scope, the outer function's scope, and the global scope, allowing it to "remember" variables even after the outer function has finished executing.

Explain the purpose of the "this" keyword in JavaScript.

Answer: "this" refers to the current execution context and can change its value based on how a function is called. It's commonly used to access object properties within methods.

What is event delegation in JavaScript, and why is it useful?

Answer: Event delegation is a technique where a single event handler is attached to a common ancestor of multiple elements. It's useful for improving performance and simplifying event handling in large DOM structures.

How do you add an element to an array in JavaScript?

Answer: You can use the "push" method to add an element to the end of an array, or the "unshift" method to add it to the beginning of the array.

Explain the difference between "==" and "===" in JavaScript.

Answer: "==" checks for equality after type coercion, while "===" checks for strict equality without type coercion, meaning both value and type must match.

What is the purpose of the "JSON" object in JavaScript?

Answer: The "JSON" object is used for parsing and stringifying JSON (JavaScript Object Notation) data, which is a lightweight data interchange format.

What is the difference between "map" and "forEach" in JavaScript?

Answer: Both "map" and "forEach" are used to iterate over arrays, but "map" returns a new array with the results of the provided function, while "forEach" simply iterates without returning a new array.

Explain the "prototype" in JavaScript and its relation to objects.

Answer: The "prototype" is an object that is associated with every function and object in JavaScript. It allows objects to inherit properties and methods from other objects, creating a prototype chain.

What is the "event loop" in JavaScript?

Answer: The event loop is a core concept in JavaScript's concurrency model. It's responsible for managing the execution of code, handling asynchronous operations, and responding to events like user interactions.

How do you create an object in JavaScript?

Answer: You can create an object using object literals like this: ```javascript const myObject = { key1: 'value1', key2: 'value2', }; ```

What is a callback function in JavaScript?

Answer: A callback function is a function passed as an argument to another function, and it's executed after the completion of the parent function. Callbacks are often used for handling asynchronous operations.

Explain the concept of "promises" in JavaScript.

Answer: Promises are objects representing the eventual completion or failure of an asynchronous operation. They provide a more structured way to handle asynchronous code compared to callbacks.

What is the "localStorage" in JavaScript, and how is it used?

Answer: "localStorage" is a web storage feature that allows you to store key-value pairs in a web browser. It's often used for storing data locally on a user's device.

Explain the concept of "asynchronous programming" in JavaScript.

Answer: Asynchronous programming is a programming pattern that allows multiple tasks to be executed concurrently without blocking the main thread. It's essential for tasks like fetching data from the web or handling user input.

How do you handle errors in JavaScript using "try...catch"?

Answer: You can use a "try...catch" block to catch and handle exceptions in JavaScript. Code within the "try" block is executed, and if an error occurs, it's caught and processed in the "catch" block.

Explain the difference between "null" and "undefined" in JavaScript.

Answer: "null" is an explicit value representing the absence of any object value, while "undefined" indicates that a variable has been declared but hasn't been assigned a value.

What is the "RESTful" architecture in web development?

Answer: RESTful is an architectural style for designing networked applications. It's based on a set of principles, including stateless communication, using HTTP methods, and representing resources as URLs.

What are arrow functions in JavaScript, and how do they differ from regular functions?

Answer: Arrow functions are a more concise way to write function expressions in JavaScript. They have a shorter syntax and inherit the "this" value from their surrounding code, making them useful in certain contexts.

What is the purpose of "AJAX" in web development?

Answer: AJAX (Asynchronous JavaScript and XML) is a set of web development techniques that allow web pages to fetch data from a server without needing to refresh the entire page. It's used to create responsive and dynamic web applications.

Explain the concept of "hoisting" in JavaScript.

Answer: Hoisting is a JavaScript behavior where variable and function declarations are moved to the top of their containing scope during compilation. This means you can use variables and functions before they're declared in the code.

What is the "DOM" in web development, and how is it used?

Answer: The DOM (Document Object Model) is a representation of the structure and content of a web page, created by the browser. It allows developers to access and manipulate HTML and XML documents using programming languages like JavaScript.

What are "async" and "await" in JavaScript, and how are they used for handling asynchronous code?

Answer: "async" and "await" are used for handling asynchronous code in a more synchronous style. "async" is used to declare that a function returns a promise, and "await" is used to pause the execution until the promise is resolved or rejected.

Explain the concept of "destructuring" in JavaScript.

Answer: Destructuring is a way to extract values from objects or arrays into variables. It provides a concise syntax for assigning multiple variables at once, making code more readable and efficient.

What is the purpose of the "fetch" API in JavaScript?

Answer: The "fetch" API is used for making network requests in JavaScript. It provides a more modern and flexible way to interact with remote resources, such as fetching data from a server.

Explain the concept of "cross-origin resource sharing" (CORS) in web development.

Answer: CORS is a security feature that allows or restricts web pages to request data from a different domain. It's used to prevent unauthorized access to resources and protect user data.

What is the "spread operator" in JavaScript, and how is it used?

Answer: The spread operator (three dots: "...") is used to spread the elements of an iterable (e.g., an array) into a new array, object, or function call. It's a handy tool for copying, merging, or passing elements as function arguments.
